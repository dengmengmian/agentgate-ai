Codex + OpenRouter

通过 AgentGate 让 Codex 使用 OpenRouter。

AgentGate 给 Codex 一个本地入口，你可以在路由里选择 OpenRouter、DeepSeek、OpenAI 或其他上游。

直接答案

适合场景

  • 你想让 AI 应用保持一个稳定的本地入口。
  • 你需要切换 Provider、查看请求日志或统计成本。
  • 你希望客户端配置可以安全恢复。

不适合场景

  • 你只用一个客户端、一个 Provider，且配置长期不变。
  • 你不需要日志、故障转移、成本统计或配置回滚。

为什么不要每次都改 Codex

测试不同 Provider 时，让 Codex 指向一个稳定的本地网关更省心。AgentGate 保持本地入口不变，具体上游由路由策略决定。

接入路径

创建或启用 OpenRouter Provider，填写 API key，选择模型，然后在 AgentGate 里对 Codex 执行 apply。请求进入后，可以在日志里看到实际上游。

路由灵活性

你可以把 OpenRouter 设为默认路由，也可以添加故障转移 Provider，或者为不同客户端和任务创建不同路由策略。

适合搜索

让 Codex 指向 AgentGate，再从一个本地网关把请求路由到 OpenRouter 或其他 OpenAI 兼容 Provider。

也能接其他 OpenAI 兼容 Provider 吗？

可以。AgentGate 的目标就是用一个本地网关管理多个上游 Provider。

之后还需要再改 Codex 吗？

通常不用。Codex 保持指向 AgentGate，后续只在 AgentGate 里改路由。
