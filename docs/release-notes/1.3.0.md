## AgentGate 1.3.0 — When a connection fails, you know exactly why / 连接失败时，一眼知道为什么、怎么修

### What's new
- **Failures that tell you what to do.** A failed test connection no longer dumps a bare `HTTP 401`. 13 common failure types each get a one-line cause + suggestion, and 11 major providers get one-click buttons like "regenerate key", "check balance" or "enable plugin" that jump straight to the right console.
- **Quick setup spots the key in your clipboard.** On entering quick setup, AgentGate quietly reads the clipboard once; if it looks like a known key, a banner offers to fill it in and auto-picks the provider type. If not, it stays out of your way.
- **See the process after applying a client config.** After applying to Codex / Claude Code / OpenCode / Gemini CLI / AtomCode, the dialog shows the config path and process status — with PID and a one-click `kill <pid>` if it's running. It never kills anything for you.

### 本次更新
- **失败会告诉你怎么办。** 测试连接失败不再裸 `HTTP 401`。13 种典型失败各给一句话原因 + 建议，11 家主流 provider 直接给「去重建 key」「查看账户余额」「去开通插件」等一键按钮，跳到对应控制台。
- **快速配置自动识别剪贴板里的 key。** 进入快速配置时悄悄读一次剪贴板，识别为已知 key 就弹 banner 帮你填入并自动选好 provider type；读不到或不是 key 就完全不打扰。
- **应用客户端配置后看到进程。** 应用到 Codex / Claude Code / OpenCode / Gemini CLI / AtomCode 后，弹窗显示配置 path + 进程状态，正在跑就给 PID + 一键复制 `kill <pid>`。从不自动 kill。

### Why it matters / 为什么重要
This release is about **removing the first-five-minutes friction**: clearer failures, smarter onboarding, and visible process state turn a fiddly setup into something users can finish on their own. 这一版主打 **消除前五分钟的摩擦**：更清晰的失败、更聪明的上手、可见的进程状态，让原本难搞的配置变成用户能自己跑通的事。
