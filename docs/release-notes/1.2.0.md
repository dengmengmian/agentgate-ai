## AgentGate 1.2.0 — Smarter provider setup, less guesswork / 更聪明的供应商配置，少踩坑

### What's new
- **MiMo picks the right domain by key type and region automatically.** `sk-*` keys go to the open API, `tp-*` keys go to token-plan-{cn|sgp|ams}, avoiding 401s from mismatched key, Chat host and Anthropic host.
- **MiMo / DeepSeek recommended model mappings fill themselves in.** Creating a provider, fetching models, or applying config auto-completes `gpt-*` → upstream model and `claude-*` → upstream model mappings.
- **The provider form is now a clean three-step flow.** Basics → Models & capabilities → Advanced (collapsed by default), so newcomers aren't hit with 8+ fields at once.
- **New providers auto-pick the latest model.** Background fetch of upstream models → capability detection → choose default and reasoning models by version/tier, no hardcoded model names.
- **Provider cards show direct-connect protocol chips.** "Direct Chat" and "Direct Anthropic" make it obvious which clients connect directly without protocol conversion.

### 本次更新
- **MiMo 按 key 类型和区域自动选域名。** `sk-*` 走开放 API，`tp-*` 走 token-plan-{cn|sgp|ams}，避免 key、Chat host、Anthropic host 不匹配 401。
- **MiMo / DeepSeek 推荐模型映射自动补齐。** 创建 provider、拉模型、应用配置时自动补齐 `gpt-*` → 上游模型 / `claude-*` → 上游模型 的映射。
- **Provider 表单三段重构。** 基础 → 模型与能力 → 高级（默认折叠）。新手不再被 8+ 字段平铺吓住。
- **新建 provider 后自动挑最新模型。** 后台自动拉上游模型 → 识别能力 → 按版本/层级选 default + reasoning model，不 hardcode 模型名。
- **Provider 卡片直连协议 chip。** 「直连 Chat」「直连 Anthropic」一眼看出哪些客户端走直连不需要协议转换。

### Why it matters / 为什么重要
This release turns provider onboarding from a manual chore into a guided, self-configuring flow. 这一版把添加供应商从手动填表变成引导式、自动补齐的流程，同时收敛了直连模型解析、连接测试卡死、Codex 自检误报等老问题，整体更稳。
