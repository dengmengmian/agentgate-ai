## AgentGate 1.3.5 — Image requests route where they can actually run / 带图请求只去能跑的供应商

### What's new
- **Vision routing is now consistent across all three entry points.** Previously only `/v1/responses` skipped providers that don't support vision when a request carried an image; `/v1/chat/completions` and `/v1/messages` didn't, so image requests could be routed to a provider that was guaranteed to fail. Now all three skip non-vision providers for image requests, and failover automatically picks a vision-capable candidate.
- **More accurate usage and updates.** Token stats now fall back to `input_tokens` when the upstream returns a `null` `prompt_tokens`, the auto-update endpoint points at the real repo, and a failed update now shows a clear error instead of silently doing nothing.

### 本次更新
- **三个入口的视觉路由对齐。** 此前只有 `/v1/responses` 会在请求带图时跳过不支持视觉的供应商，`/v1/chat/completions` 和 `/v1/messages` 不会，导致带图请求可能被路由到必然失败的供应商。现在三个入口统一：带图时跳过不支持视觉的供应商，失败转移自动选到支持视觉的候选。
- **统计与更新更准。** 上游返回 `prompt_tokens` 为 `null` 时 token 统计正确回退到 `input_tokens`；自动更新指向真实仓库；更新失败会明确报错而不是无声清空。

### Why it matters / 为什么重要
A correctness-focused release: requests land where they can succeed, usage numbers are trustworthy, and update failures stop hiding — all of which reduce silent breakage. 一个偏正确性的版本：请求落到能成功的地方、用量数字可信、更新失败不再隐身，都在减少静默故障。
