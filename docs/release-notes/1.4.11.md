## AgentGate 1.4.11 — Clearer consoles & pet chat / 更清晰的控制台与宠物聊天

### What's new
- **Pet Chat now has a real home.** The main app includes a dedicated Pet Chat page that shares the desktop pet's conversation history, personality, memory, and gateway chat path.
- **Major pages are easier to read.** Overview, Providers, Provider Detail, Routes, Gateway, Diagnostics, Logs, MCP, Skills, Global Instructions, Clients, Settings, and Pet Chat now use a more consistent console layout with clearer status panels and primary work areas.
- **Pet Chat stays compact during long sessions.** The conversation stream scrolls inside its panel, and memory editing now lives beside the send box instead of taking a full row.
- **The desktop pet is much smarter.** Greetings, the periodic usage report, error explanations, and occasional check-ins are now generated by a real model (with graceful fallback to local lines when the gateway is off), and the pet remembers preferences you mention in chat.
- **The pet feels more alive.** New reactions: a spinning badge while Claude Code works plus a celebration when it finishes, bounce intensity that scales with gateway load (with a sweat drop under heavy load), a temper when poked repeatedly, a "stuffed" look when today's spend is high, seasonal / late-night / Friday accessories, and edge-docking when dragged to the screen border.
- **Editable pet memory.** The Pet Chat page lets you view and edit what the pet remembers; memory and chat history now clear independently and stay in sync across the pet window and the page.

### Fixes
- **Disabled providers stay disabled.** Route candidate selection now checks both the route-profile provider flag and the provider's own `enabled` flag, so disabled providers are no longer picked by routing.
- **Pet chat works behind a local proxy.** The pet's gateway requests now bypass the system/HTTP proxy, fixing the empty `HTTP 502` seen when a local proxy (e.g. Clash) was running.
- **Pet chat no longer breaks on strict providers.** Dropped the fixed `temperature` some models reject, and guard against empty assistant turns that made providers reject the next message.

### 本次更新
- **宠物聊天有了正式页面。** 主窗口新增宠物聊天页,与桌宠共享聊天记录、人格、记忆和网关聊天链路。
- **主要页面更容易读。** 概览、供应商、供应商详情、路由、网关、诊断、日志、MCP、技能、全局指令、客户端、设置和宠物聊天统一为更清晰的控制台式布局,状态面板和主工作区层级更明确。
- **长会话不再把页面撑长。** 宠物聊天的会话流在面板内部滚动,记忆编辑移到发送框旁边,不再单独占一整行。
- **桌宠更聪明了。** 开机问候、定时用量播报、报错解释、偶尔的主动搭话现在都由真实模型生成(网关不可用时回落本地文案),还能记住你在聊天里提到的偏好。
- **桌宠更有生命力。** 新增反应:Claude Code 干活时头顶转徽章、完成时跳跃庆祝;弹跳强度随网关忙闲分级(高负载冒汗);连戳会生气;今日花费超标会"吃撑"变圆;节日 / 深夜 / 周五会换小装饰;拖到屏幕边缘会贴边探头。
- **宠物记忆可编辑。** 宠物聊天页可以查看和编辑宠物记住的信息;记忆和聊天记录现在分开清空,并在宠物窗口与页面之间实时同步。

### 修复
- **禁用供应商不会再被路由选中。** 路由候选现在同时检查路由档位供应商启用状态和供应商自身 `enabled` 标记,禁用供应商不会进入候选。
- **宠物聊天在本机代理下也能用。** 宠物发往网关的请求现在绕过系统/HTTP 代理,修复本机开着代理(如 Clash)时出现的空 `HTTP 502`。
- **宠物聊天不再被严格供应商卡住。** 去掉部分模型拒绝的固定 `temperature`,并防止空 assistant 消息导致下一条请求被供应商拒绝。

### Why it matters / 为什么重要

This release is mostly about making AgentGate calmer to operate: dense pages explain themselves faster, routing state is harder to misread, and the pet chat can grow naturally without breaking the page layout. 这一版主要让 AgentGate 更好管理:信息密集页更快看懂,路由状态更不容易误判,宠物聊天可以自然变长但不破坏页面布局。
