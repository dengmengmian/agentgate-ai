## AgentGate 0.1.9 — Native Claude support for Codex / 让 Codex 原生跑通 Claude

### What's new
- **Native Claude Messages API support.** When a provider is `anthropic`, Codex requests are converted to Claude's native Messages format, with full `tool_use` / `tool_result` / `input_schema` / `thinking.budget_tokens` support.
- **URL-driven routing.** A new `responses_base_url` lets AgentGate decide per request whether to pass through, convert to Claude, or convert to Chat — and it understands both full URLs and base URLs.
- **More providers and cleaner labels.** Adds Anthropic and MiniMax provider types, with friendly protocol labels on each card.
- **Fewer Codex hangs and quirks.** No more meaningless `thinking` fields for DeepSeek, no hangs on empty non-streaming responses, and tool call IDs are trimmed to fit the Responses API.

### 本次更新
- **原生 Claude Messages API 支持。** provider 为 `anthropic` 时，Codex 请求会转成 Claude 原生 Messages 格式，完整支持 `tool_use` / `tool_result` / `input_schema` / `thinking.budget_tokens`。
- **URL 驱动的路由。** 新增 `responses_base_url`，AgentGate 按请求决定走透传、Claude 转换还是 Chat 转换，并且填完整 URL 或 base URL 都能识别。
- **更多 provider 和更清晰的标签。** 新增 Anthropic / MiniMax provider 类型，卡片上显示友好的协议标签。
- **Codex 更少卡顿和怪问题。** DeepSeek 不再收到无意义的 `thinking` 字段，非流式空响应不再挂起，tool call ID 也会按 Responses API 限制截断。

### Why it matters / 为什么重要
This is the version where AgentGate stops being OpenAI-shaped and starts speaking each upstream's native protocol. 从这一版起，AgentGate 不再只贴合 OpenAI 形态，而是开始用各家上游的原生协议对话。
