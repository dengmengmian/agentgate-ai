## AgentGate 1.3.6 — Cleaner history, steadier under load / 历史可清理，高并发更稳

### What's new
- **Delete old config history entries.** Both client config history and global instruction (CLAUDE.md / AGENTS.md) history can now be removed one by one instead of piling up forever. The "initial" snapshot stays protected so you can always roll back to your pre-AgentGate setup.
- **Steadier under concurrent load.** The gateway moved to a connection pool, so multiple concurrent requests no longer queue behind a single global lock. You won't notice it day to day, but high QPS and heavy logging stay responsive.

### 本次更新
- **配置历史可逐条删除。** 客户端配置历史和全局指令（CLAUDE.md / AGENTS.md）历史都能逐条删除，不再一直堆叠。「初始」快照受保护、不可删，随时能回滚到接入 AgentGate 前的配置。
- **高并发更稳。** 网关换成连接池，多个并发请求不再排在一把全局锁后面；日常无感，QPS 高或日志量大时响应更稳。

### Why it matters / 为什么重要
A maintenance-focused release: it removes a long-standing serialization bottleneck and lets users keep their config history tidy — both improve trust for heavier, longer-term usage. 一个偏维护的版本：解除了长期存在的串行瓶颈，并让用户能清理配置历史，二者都提升了重度、长期使用下的可靠感。
