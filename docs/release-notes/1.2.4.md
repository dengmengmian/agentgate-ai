## AgentGate 1.2.4 — Switch providers without breaking your client / 换 provider 不再被旧模型名坑

### What's new
- **AtomCode / OpenCode one-click setup now survives provider switches.** The client is wired to the `agentgate` virtual model instead of the current provider's real model name, so switching providers no longer leaves you with a stale model name that triggers 400 errors. Real upstream model names still pass through via your Model Mapping rules.

### 本次更新
- **AtomCode / OpenCode 一键配置换 provider 不再报错。** 客户端绑定到 `agentgate` 虚拟模型，而不是把当前 provider 的真实模型名固化进去，切换 provider 后不会带着旧模型名 400。原生真实模型名仍按 Model Mapping 规则透传。

### Why it matters / 为什么重要
This removes a sharp edge in multi-provider workflows: 一键配置一次，后续随意切换 provider 都不用重配客户端。
